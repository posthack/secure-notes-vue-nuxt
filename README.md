# Secure Notes

Заметки со сквозным шифрованием на клиенте. Мастер-пароль не уходит с устройства, из него в браузере выводятся ключи, на сервер попадает только шифртекст. Прочитать данные сервер не может даже теоретически.

Пет-проект. Цель не в количестве фич, а в честной криптоархитектуре: правильная модель ключей и внятная threat model.

## Как устроена приватность

- Мастер-пароль → PBKDF2 (600k итераций, соль на пользователя) → мастер-ключ. Argon2id был бы лучше, но он тянет WASM-зависимость — остался PBKDF2 как осознанный компромисс.
- Мастер-ключ не шифрует заметки напрямую. Он оборачивает (AES-KW) случайный ключ на каждый элемент (DEK). Смена пароля перешифровывает только обёртки, а не все данные.
- Каждая заметка шифруется AES-256-GCM со своим IV; id заметки идёт в AEAD, так что подмена метаданных ломает расшифровку.
- Ключи держатся в памяти веб-воркера, не в `localStorage`; при выходе стираются.

## Синхронизация

Аккаунт (Better Auth, email+пароль) нужен только чтобы синкать между устройствами. Пароль аккаунта и мастер-пароль — разные вещи: сервер хранит лишь зашифрованные blob'ы и мастер-пароль не видит. Синк — last-write-wins по времени правки.

## Стек

- Nuxt 4 + Vue 3 + TypeScript, Nuxt UI, Pinia
- Nitro + Drizzle + libsql (сервер хранит только шифртекст)
- Better Auth
- Web Crypto API, вычисления в веб-воркере
- Vitest

## Запуск

```bash
npm install
cp .env.example .env   # проставить BETTER_AUTH_SECRET
npm run db:migrate     # накатить схему в локальный sqlite
npm run dev            # дев-сервер, http://localhost:3000

npm run build          # production-сборка
npm test               # тесты
npm run typecheck      # проверка типов
npm run lint           # линт
```
