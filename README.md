# Secure Notes

Заметки и файлы со сквозным шифрованием на клиенте. Мастер-пароль не уходит с устройства, из него в браузере выводятся ключи, на сервер попадает только шифртекст. Прочитать данные сервер не может даже теоретически.

Пет-проект. Цель не в количестве фич, а в честной криптоархитектуре: правильная модель ключей и внятная threat model.

## Как устроена приватность

- Мастер-пароль → PBKDF2 (соль на пользователя) → мастер-ключ. Argon2id был бы лучше, но он тянет WASM-зависимость; почему остался PBKDF2 — в `SECURITY.md`.
- Мастер-ключ не шифрует заметки напрямую. Он оборачивает случайный ключ на каждый элемент (DEK). Смена пароля перешифровывает только обёртки, а не все данные.
- Каждая заметка и файл шифруются AES-256-GCM со своим IV; метаданные аутентифицируются через AEAD.
- Ключи живут в памяти, не в `localStorage`. Есть auto-lock и стирание при выходе.

## Стек

- Nuxt 4 + Vue 3 + TypeScript
- Nitro + Drizzle (сервер хранит только blob'ы шифртекста)
- Web Crypto API
- Vitest

## Запуск

```bash
npm install
npm run dev       # дев-сервер, http://localhost:3000
npm run build     # production-сборка
npm test          # тесты
npm run typecheck # проверка типов
npm run lint      # линт
npm run format    # форматирование
```
